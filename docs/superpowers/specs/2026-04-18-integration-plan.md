# DocSeeker 功能整合规划

> 维护人：lizhgb
> 更新日期：2026-04-18
> 状态：进行中

---

## 一、背景

DocSeeker 已完成数据库分片架构重构，核心搜索功能稳定。为提升用户体验和技术质量，整合以下三大模块：

1. **系统集成** - 让 DocSeeker 成为合格的后台常驻应用
2. **搜索体验** - 优化搜索交互细节
3. **后台优化** - 首次运行引导、智能缓存、VACCUM 压缩

---

## 二、整体架构

**新增设置页面**（`SettingsPage.tsx`）作为所有开关的统一入口：
- **外观 Tab**：主题选择 + 跟随系统开关
- **窗口行为 Tab**：开机自启 + 静默启动 + 关闭最小化托盘

**实施方式**：渐进增强，短周期、小改动、频繁提交。

---

## 三、功能清单

### 模块 A：系统集成

| 功能 | 说明 | 依赖 |
|------|------|------|
| 开机自启 | `app.setLoginItemSettings()` | 无 |
| 静默启动 | `--startup` 命令行参数检测 | 无 |
| 关闭最小化托盘 | 可配置的关闭行为 | 托盘已存在 |
| 托盘菜单增强 | 显示窗口/全局搜索/退出 | 托盘已存在 |
| 跟随系统主题 | `nativeTheme` API 监听系统主题 | 无 |

### 模块 D：搜索体验

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 搜索 debounce | 300ms 延迟，避免每次击键搜索 | P0 |
| 结果中二次过滤 | 结果列表上方添加筛选输入框 | P0 |
| 隐藏文件开关 | 扫描设置中增加 includeHidden | P1 |
| 系统文件开关 | 扫描设置中增加 includeSystem | P1 |

### 模块 E：后台优化

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 首次运行引导 | 自动添加 Documents/Desktop 到扫描列表 | P0 |
| VACUUM 压缩 | 定期压缩 shard，回收已删除记录空间 | P2 |
| 热点缓存策略 | 基于访问频率而非最近搜索 | P2 |

## 五、暂缓功能（后续迭代）

以下功能技术难度较高或优先级较低，暂不纳入本次规划：

- 实时监控（USN Journal）- 需改用 NTFS USN Journal API
- 多格式预览组件（代码高亮/图片缩放/PDF/Office）
- 搜索结果缩略图预览
- 右键资源管理器集成

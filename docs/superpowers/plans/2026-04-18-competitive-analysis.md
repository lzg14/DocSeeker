## 四、核心差距说明

### 4.1 实时文件监控（Phase 3 优先）

**现状：** 用户扫描目录后，新增或修改的文件无法被搜索，必须手动执行增量扫描。

**竞品对比：** Everything / AnyTXT / Copernic 均支持文件变更即时感知，无需用户干预。

**解决方案：** Windows NTFS USN Journal
- 读取文件系统变更日志，无需轮询，CPU 开销接近零
- 毫秒级感知文件创建/修改/删除
- macOS: FSEvents / Linux: inotify（跨平台预留）

### 4.2 多格式文件预览（Phase 3 优先）

**现状：** 点击搜索结果只显示前 500 字纯文本，不够"桌面应用"。

**竞品对比：** AnyTXT / Copernic 提供缩略图预览，Listary 提供快速预览面板。

**解决方案：** 按文件类型提供原生预览
- 图片：直接渲染（img 标签）
- PDF：iframe 嵌入或 pdf-parse 渲染
- Markdown：实时渲染为 HTML
- 代码：highlight.js 语法高亮
- 已有内容提取器（mammoth/xlsx/pdf-parse）直接复用

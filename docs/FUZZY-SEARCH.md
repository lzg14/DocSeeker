# 模糊搜索实现方案

> 更新时间: 2026-04-22
> 目标: 实现打字容错搜索，打错字也能找到目标

---

## 背景

目前 DocSeeker 使用 SQLite FTS5 进行精确搜索，用户必须输入正确的关键词才能匹配。如果输入有拼写错误（如 "documnet" 代替 "document"），将搜索不到任何结果。

模糊搜索可以容忍一定的字符差异，让用户即使打错字也能找到目标文件。

## 方案选择

### 方案对比

| 方案 | 原理 | 优点 | 缺点 | 适合场景 |
|------|------|------|------|----------|
| **Fuse.js 后置** | 精确搜索 + 结果模糊排序 | 简单、性能好 | 牺牲查全率 | ✅ **推荐** |
| N-gram 索引 | 字符片段匹配 | 查全率高 | 索引体积 +50% | 大数据量 |
| Trigram | SQLite 扩展 | 数据库原生 | 需要安装扩展 | 混合方案 |

### 推荐方案: Fuse.js 后置过滤

**原理**:
1.先用 FTS5 精确搜索，返回 TOP 1000 条结果
2.用 Fuse.js 对这 1000 条做模糊排序
3.返回按相似度排序的结果

**优势**:
- 不影响现有 FTS5 索引
- 搜索结果少，Fuse.js 毫秒级完成
- 用户体验：先精确匹配，再模糊排序

## 实现计划

### 1. 安装依赖

```bash
npm install fuse.js
```

### 2. 修改搜索流程

```
用户输入 "documnet"
       ↓
FTS5 精确搜索 → 返回 100-1000 条结果
       ↓
Fuse.js 模糊排序 → 返回按相似度排序的结果
       ↓
展示搜索结果
```

### 3. Fuse.js 配置

```typescript
import Fuse from 'fuse.js'

const fuseOptions = {
  keys: ['name', 'content_snippet'],
  threshold: 0.4,        // 0.0 = 精确, 1.0 = 任意
  distance: 100,          // 字符距离
  includeScore: true,    // 返回相似度分数
  ignoreLocation: true,  // 忽略位置
  useExtendedSearch: true
}
```

### 4. UI 变化

| 元素 | 变化 |
|------|------|
| 搜索框 | 添加"模糊搜索"开关按钮 |
| 搜索结果 | 显示相似度分数 |
| 无结果时 | 提示"开启模糊搜索" |
